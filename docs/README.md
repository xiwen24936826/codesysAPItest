# Docs

项目文档按以下维度组织：

- `architecture/`: 仓库结构、模块边界、服务分层说明
- `api_specs/`: MCP 工具与内部服务接口规范
- `research/`: CODESYS API 调研、能力映射、供应商差异记录
- `codex_client_handbook.md`: 面向 Codex 客户端的轻量协作手册
- `codex_client_handbook.md`: 当前主客户端使用规则文档
- `mcp_client_connection.md`: 统一的 MCP 客户端连接方式说明
- `claude_code_connection.md`: Claude Code 连接与最小启动说明
- `claude_code_troubleshooting.md`: Claude Code 常见失败排查表
- `git_workflow.md`: Git 与 GitHub Desktop 工作说明
- `current_phase_plan.md`: 当前阶段计划与稳定边界
- `architecture/server_assembly.md`: 本地 server/tool/client 装配说明

当前建议优先阅读顺序：

1. `current_phase_plan.md`
2. `codex_client_handbook.md`
3. `claude_code_connection.md` 或 `mcp_client_connection.md`
